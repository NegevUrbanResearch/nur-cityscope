Layout / safe area only — no new capabilities

Layers model (OTEF behavior — annotations for implementers): Packs in the app come from the effective layer groups at runtime and may include curated special handling (e.g. merge). Some list rows can represent several full layer ids — without noisy processed/curated/raw-id badges in the UI; a compact flow-category indicator (e.g. זרם) is allowed when it clarifies semantics. A row’s visibility toggle and its animation control are independent where the layer supports animation. Flow packs retain pack-level Flow control in the focused pack header (current-behavior parity); row-level animation chips are separate. The mockup shows mixed per-row animation in the adjacent זרימה focused phone. Overview uses a pack master toggle separate from the open/drill control. Overflow: if pack tiles or focused rows exceed the visible area, scroll inside the tab/panel container — the mockup avoids implying a single endless scroll of all layers at once. Table switcher and full connection copy are omitted from these mockup frames on purpose; only a green/red lamp is shown (offline = status-pip is-off in a future comp).
